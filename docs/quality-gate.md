# Hero Pro 质量闸门

## 目标

- 保证进入生产环境的英雄画像、海克斯画像与推荐结果具备最低可信度。
- 保证正式推荐闸门与数据审计闸门完全一致。
- 保证每次规则调整后，都能通过固定基准样例检查推荐稳定性。

## 一、机制数据上线门槛

### 英雄画像

必须满足：

- `verification.status = verified`
- 必须存在 `riot-ddragon` 证据
- 技能机制字段完整
- 标签字段可解释，且能回溯到技能或官方描述
- `scalingTags / triggerTraits / survivalStyles / utilityTags` 与技能机制不能明显冲突

禁止上线：

- 只有社区评级，没有官方文本
- 标签结论与技能描述对不上
- 版本不明或来源不可复查

### 海克斯画像

必须满足：

- `verification.status = verified`
- 名称、稀有度、正式描述、图标来源已确认
- 触发方式与收益类型可解释
- 风险标签已填写或明确无显著风险
- `id` 必须存在于 `src/data/augments/allowlist.ts`
- 必须同时具备：
  - `official-client`
  - `riot-support`
  - `community-mirror`

Arena 强化符文的生产验证，允许以下组合通过：

- Riot 官方 Arena 模式说明
- 客户端导出文本镜像 `cdragon/arena/*.json`
- 条目清单镜像 `cherry-augments.json`

额外要求：

- 条目 `id / name / rarity / icon` 一致
- 描述中的关键数值可直接读取或由同条目数据字段还原
- 若客户端导出文本和社区页面有冲突，以客户端导出文本为准
- `cdragon/arena/*.json` 仅代表原始客户端导出池，不代表生产可收录池
- `preferredUsers / positiveSignals / trapSignals` 需要能解释触发与收益边界
- 含 `数值`、模板残句或未还原占位的描述不得进入 `verified`

禁止上线：

- 只有社区站点结论，没有官方或客户端文本支撑
- 正式描述与触发逻辑冲突
- 中英文名映射不稳定
- 条目不在 Arena production allowlist 中

## 二、推荐结果验收

每条正式推荐至少需要满足：

- 有总分
- 有结论级别
- 有分维度拆解
- 有推荐理由
- 有风险提示或明确无显著风险

必须避免：

- 只给排序，不给原因
- 结论与分解理由互相矛盾
- 未验证条目混入正式推荐
- 明显不适配组合被长期推到前列

### 推荐稳定性闸门

每次调整 `rules.ts` 或新增关键画像后，必须通过：

- `npm run audit:recommendations`

该回归校验用于固定一组方向明确的基准样例，当前至少覆盖：

- `Lulu` 更偏 `All For You`
- `Vayne` 更偏 `Typhoon`
- `Swain` 更偏 `Big Brain`
- `Leona` 不应把 `Skilled Sniper` 排到第一
- `Lux` 更偏 `Skilled Sniper`

说明：

- 这不是完整真值系统，而是推荐稳定性闸门
- 如果样例断言失败，必须先解释是规则误判还是基准集需要升级，再决定是否放行

## 三、页面验收

首页与查询页必须满足：

- 首屏能说明产品用途
- 首屏能说明为什么可信
- 主要操作路径不超过 3 步
- 单屏不堆砌大量说明文字
- 重要信息优先于补充信息

结果页必须满足：

- 第一眼能看到推荐顺序
- 第二眼能看到推荐理由
- 第三眼能看到风险与适用场景

必须避免：

- 用户需要滚动很多屏才能找到答案
- 同时展示过多技术术语
- 页面视觉重点分散

## 四、开发阶段的最小校验

每次涉及数据、规则或验证逻辑的改动后，至少补跑：

- `npm install`
- `npm run audit:data`
- `npm run audit:recommendations`
- `npm run lint`
- `npm run build`

若未来继续扩大基准集，还需要补充：

- 画像结构一致性校验
- 推荐结果断言扩充
- 关键手工组合抽样复核

### 自动化边界

- 允许自动化的范围：
  - 事实层同步
  - 草稿模块生成
  - 证据骨架预填
- 不允许自动化直接决定：
  - 最终画像标签
  - `verified` 放行
  - 正式推荐索引接入

说明：

- 自动生成的录入审查包只是一种提效工具，不是质量豁免。
- 任何自动生成草稿在进入生产前，仍必须通过 `audit:data`、`audit:recommendations` 与人工抽样复核。
- 复盘结论：不能再把 Arena 原始导出池直接当成生产海克斯池；allowlist 是新增的范围闸门。

## 五、当前结论

- 当前项目已具备来源规范、类型结构、数据审计、推荐回归和生产校验闸门。
- 当前重点不在 UI，而在持续扩样本时保持数据可信、规则稳定、误判可追踪。
