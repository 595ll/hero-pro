# 第十三批继续录入计划

## Summary

- 本轮继续只扩英雄生产数据，不新增生产海克斯。
- 目标批次收紧为 2 个英雄：`Aatrox`、`KogMaw`。
- 选择原则：优先补当前生产层仍相对稀缺、但可被现有 20 条 allowlist 海克斯和现有评分规则稳定解释的 archetype。
- 成功标准：
  - 新增英雄画像满足 `verified` 证据要求与字段完整性要求；
  - 不修改现有海克斯 allowlist；
  - `audit:data`、`audit:recommendations`、`lint`、`build` 全部通过；
  - 不引入查询页破图或推荐回归。

## Proposed Changes

### 1. 新增第十三批英雄数据文件

- 新建 `src/data/champions/thirteenth-batch.ts`
- 录入对象：
  - `Aatrox`
  - `KogMaw`
- 录入要求：
  - 证据统一引用 `Data Dragon zh_CN / en_US`
  - 补齐 `damagePatterns / combatTempos / scalingTags / triggerTraits / survivalStyles / utilityTags`
  - 每个英雄补齐 3 条 `coreMechanics`、至少 1 条 `antiPatterns`
  - 5 个技能条目都必须有可解释的 `mechanics`
- 画像方向：
  - `Aatrox`
    - 定位为近战持续站场斗士，不写成纯坦克
    - 重点体现吸血、Q 连段、贴身回复与长回合压制
    - 目标是让 `Goredrink / Ocean Soul / Perseverance` 都具备明确兑现路径
  - `KogMaw`
    - 定位为纯 on-hit / 高攻速远程炮台，不写成泛化远程输出
    - 重点体现 W 强化普攻、持续平 A、长回合站桩输出与吃保护需求
    - 目标是让 `Typhoon / Light 'em Up!` 形成比泛用被动海克斯更清晰的正向样本

### 2. 更新英雄生产入口

- 修改 `src/data/champions/index.ts`
- 接入 `thirteenthBatchChampionProfiles`
- 新批次放在数组前部，保持与最近批次一致的组织顺序

### 3. 扩充推荐回归样例

- 修改 `scripts/audit-recommendations.mjs`
- 新增至少 2 组回归：
  - `Aatrox` 生存/站场基线
    - 候选集应包含 `138`、`64`、`60`
    - 目标是验证近战吸血斗士不会被误导到明显错位的海克斯
  - `KogMaw` on-hit 基线
    - 候选集应包含 `87`、`51`，并搭配至少 1 条泛用输出海克斯作对照
    - 目标是验证纯 on-hit 炮台会优先命中 `Typhoon` 或 `Light 'em Up!`，而不是被无关泛用项抢第一
- 回归写入原则：
  - 只固化“方向明确”的第一名或允许集合
  - 如果实测出现两个等价强解，可用 `expectedTopAugmentIds`
  - 如果第一名不够稳，宁可收紧批次、停下来复核，也不写弱断言

### 4. 不在本批实施的内容

- 不新增 `src/data/augments/*.ts`
- 不扩大 `src/data/augments/allowlist.ts`
- 不为了美术完整性强制新增本地图标资源
- 不修改 `src/lib/recommendation/rules.ts`，除非在事实录入完成后发现现有规则无法稳定解释新英雄排序；若出现这种情况，应先停止并复核，再决定是否进入规则调整

## Assumptions & Decisions

- 决策：本批只做 2 个英雄，而不是继续做 3 个以上，理由是当前用户优先级是准确性和稳态，不是纯数量。
- 决策：优先 `Aatrox` 与 `KogMaw`，因为它们分别补的是“近战吸血站场斗士”和“纯 on-hit 炮台”，都能直接复用现有 allowlist 海克斯，且比再补辅助/前排/炮台法师更有边际价值。
- 决策：本批不补新海克斯。当前生产海克斯边界已明确收紧，继续扩英雄比扩海克斯更稳。
- 决策：图标不是本批阻塞项。查询页已有安全回退，不会因为缺少 `Aatrox` / `KogMaw` 本地图标而影响功能正确性。
- 假设：`tmp/intake-batches/full-facts-intake-2026-04-25/` 中已有足够事实层草稿可复用；若执行阶段发现与 `source-cache` 最新缓存冲突，则以 `source-cache` 为准并重新核实后再录入。
- 停止条件：
  - 若 `Aatrox` 或 `KogMaw` 的推荐第一名无法形成清晰、可解释、可回归的结论，则暂停该英雄放入 `verified`，不要为了凑批次强行并入生产。

## Verification Steps

1. 录入后先跑 `npm run audit:data`
2. 再跑 `npm run audit:recommendations`
3. 再跑 `npm run lint`
4. 最后跑 `npm run build`
5. 抽样核对：
  - 查询页能看到新增英雄条目
  - 无本地图标时显示文字占位，而不是破图
  - `Aatrox` 与 `KogMaw` 的候选海克斯排序与新增回归断言一致
