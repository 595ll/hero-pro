# 英雄列表多列修正计划

## Summary

- 目标：修正当前“英雄选择仍然一行一个”的问题，让英雄列表按预期显示为多行多个。
- 成功标准：
  - 英雄选择区在桌面端显示为多列网格，而不是单列堆叠。
  - 不影响现有英雄搜索、图标展示、选中态、详情卡和推荐逻辑。
  - 移动端仍保持当前单列降级行为，不引入布局错乱。
- 约束：
  - 只做最小范围修复，不调整查询逻辑、不重做样式体系。
  - 不改数据模型、不改推荐链路。

## Current State Analysis

### 已确认现状

- 英雄列表渲染节点位于 `src/components/query-panel.tsx`：
  - 外层容器：`champion-scroll-area`
  - 列表容器：`champion-list`
  - 单项：`champion-item`
- 当前 `query-panel.tsx` 已正确使用 `filteredChampions.map(...)` 输出英雄按钮，结构本身支持多列。
- 当前 `src/app/globals.css` 中：
  - `.champion-list` 只定义了 `grid-template-columns: repeat(2, minmax(0, 1fr));`
  - 但没有定义 `display: grid`
- 同文件中 `.augment-list, .result-list` 已有 `display: grid`，说明英雄列表样式在之前调整时漏掉了这一关键声明。

### 根因判断

- 因为 `.champion-list` 没有 `display: grid`，浏览器不会应用 `grid-template-columns`。
- 结果就是英雄按钮按普通文档流逐个纵向堆叠，表现为“一行一个”。
- 这不是组件逻辑问题，也不是容器宽度不足导致，而是一个纯样式层缺项。

## Assumptions & Decisions

- 本次采用最小修复：
  - 给 `.champion-list` 显式补回 `display: grid`
  - 保留现有 `grid-template-columns: repeat(2, minmax(0, 1fr))`
- 保持当前移动端媒体查询不变：
  - 小屏继续回退为单列
- 不额外调整：
  - `champion-item` 内容结构
  - 搜索框位置
  - 详情卡
  - 海克斯区布局

## Proposed Changes

### 1. `src/app/globals.css`

- 在 `.champion-list` 规则中补充：
  - `display: grid`
- 保留现有：
  - `grid-template-columns: repeat(2, minmax(0, 1fr))`
- 检查是否存在重复或残留的 `.champion-list` / `.champion-item` 规则，避免样式冲突。

### 2. 验证现有联动不受影响

- 虽然不改 `src/components/query-panel.tsx`，但需要验证：
  - 英雄搜索后仍能在多列中显示过滤结果
  - 选中英雄后详情卡正常
  - 多列显示不影响点击选中

## Verification Steps

- 静态检查：
  - 检查 `src/app/globals.css` 无新的语法或重复选择器问题
- 页面验证：
  - 英雄列表在桌面端显示为 2 列网格
  - 输入英雄搜索词后，过滤结果仍按多列展示
  - 点击任一英雄后，选中态与详情卡正常
  - 海克斯区和结果区表现不变
- 构建验证：
  - `npm run lint`
  - `npm run build`

## Out Of Scope

- 不新增搜索或筛选功能
- 不改海克斯列表布局
- 不调整推荐算法或数据结构
- 不重构查询区组件
