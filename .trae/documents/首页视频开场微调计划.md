# 首页视频开场微调计划

## Summary
- 将首页正文的渐显时机从“视频快结束时”改为“视频播放到第 2 秒时开始渐显”。
- 在视频开场播放期间，于左下角增加一个不干扰交互的 `Hero Pro` 品牌角标，采用“品牌标识”方案：标题 + 一句简短定位文案。
- 保持现有页面结构、查询面板、滚动固定背景、自动播放失败降级与减弱动画兼容逻辑不变，确保不影响当前功能。

## Current State Analysis
- 当前首页视频逻辑集中在 `src/app/page.tsx`，通过 `videoStage` 管理 `intro / fading / background / fallback` 四种状态。
- 当前正文显示时机由 `INTRO_FADE_OUT_SECONDS = 1.25` 驱动：只有在视频距离结束不足 1.25 秒时，`handleTimeUpdate()` 才会把状态切到 `fading`。
- 当前首页没有独立的开场品牌角标，视频层只有背景视频和遮罩；`Hero Pro` 品牌信息只存在于正文首卡中。
- 当前固定背景视频与正文渐显样式集中在 `src/app/globals.css`，其中 `.video-stage` 已是 `position: fixed`，`.page-shell` 通过透明度和位移做入场过渡。
- 仓库已有基础校验命令 `npm run lint` / `npm run build`；其中 `build` 目前存在与本次首页无关的既有类型问题，计划执行时应区分“本次改动回归”与“仓库现存失败”。

## Proposed Changes
### `src/app/page.tsx`
- **What**
  - 将正文渐显触发条件改为“视频播放时间达到 2 秒”。
  - 为视频开场层增加一个左下角品牌角标节点。
  - 保留现有 `fallback`、`prefers-reduced-motion`、视频结束后切到循环背景的逻辑。
- **Why**
  - 用户希望更早看到网页内容，不再等待视频快结束。
  - 用户希望视频开场时就能感知 `Hero Pro` 品牌，但不能影响现有功能和页面阅读。
- **How**
  - 将当前“基于剩余时长”的触发方式改为“基于绝对播放时间”的触发方式，例如以 `2` 秒常量控制。
  - 在 `handleTimeUpdate()` 内，只要 `currentTime >= 2` 且仍处于 `intro`，就切换到 `fading`；避免重复触发。
  - 在视频层内新增一个仅展示型的品牌角标容器，文案采用：
    - 主标题：`Hero Pro`
    - 辅助文案：`海克斯大乱斗候选强化推荐`
  - 角标应在 `intro` 和 `fading` 阶段可见，在 `background` 与 `fallback` 阶段淡出，避免和正文首卡重复抢视觉重心。
  - 角标容器使用纯展示属性，不绑定交互，不影响现有键盘、鼠标和触屏操作。

### `src/app/globals.css`
- **What**
  - 为左下角品牌角标补充定位、层级、透明度和响应式样式。
  - 微调视频层和正文层的过渡表现，确保“第 2 秒开始渐显”后仍自然，不产生闪烁或突兀跳变。
- **Why**
  - 开场品牌信息必须稳定贴在左下角，同时不能遮挡正文和交互区域。
  - 更早开始显示正文后，需要保证视觉过渡仍顺滑。
- **How**
  - 新增类似 `.intro-brand`, `.intro-brand-title`, `.intro-brand-copy` 的样式类。
  - 角标定位在左下角安全区，使用 `position: absolute` 挂在 `.video-stage` 内，配合适度半透明背景、描边或文字阴影保证在复杂视频画面上可读。
  - 为角标设置 `pointer-events: none`，避免覆盖正文或误拦截点击。
  - 在移动端媒体查询中降低角标尺寸、缩短左右边距，避免遮挡底部内容。
  - 角标与正文分别控制淡入淡出：正文在第 2 秒后渐显，角标在正文完成进入后淡出或弱化。

### 保持不变的范围
- `src/components/query-panel.tsx` 与数据推荐逻辑不调整。
- 视频结束后转为固定背景、页面滚动时背景不跟随滚动的机制保持不变。
- 自动播放失败、视频报错、`prefers-reduced-motion` 时回退到可用静态页面的机制保持不变。
- 不处理当前仓库中与本需求无关的数据类型问题，例如 `src/data/**` 的既有 warning / build 失败项。

## Assumptions & Decisions
- “在视频播放第二秒时，渐变进入网页”解释为：**从第 2 秒开始触发正文渐显**，而不是要求在第 2 秒前完全显示。
- 左下角内容采用“品牌标识”方案，不增加按钮、链接、倒计时或其它交互元素。
- 品牌角标只服务于开场识别，不作为持久悬浮组件长期保留在正文浏览阶段。
- 所有改动限定在首页展示层，不改动数据、推荐、筛选和结果展示功能。
- 验收重点是“体验更改不破坏现有功能”，因此优先验证首屏流程、滚动背景、查询面板可操作性和降级场景。

## Verification Steps
- 代码检查
  - 读取 `src/app/page.tsx`，确认渐显触发条件改为 `currentTime >= 2` 一类的绝对时间判断。
  - 读取 `src/app/globals.css`，确认新增左下角品牌角标样式，且包含 `pointer-events: none` 与移动端适配。
- 行为验证
  - 打开首页，确认视频开始播放后左下角立即出现 `Hero Pro` 品牌角标。
  - 等待到视频第 2 秒，确认正文开始渐显，不再等到视频接近结束才进入。
  - 确认视频播放结束后仍切换为固定背景，页面滚动时背景不跟随滚动。
  - 确认查询面板仍可正常选择英雄、勾选海克斯，不被开场品牌层遮挡。
  - 确认自动播放失败或 `prefers-reduced-motion` 场景下，页面仍能直接进入可用状态。
- 诊断与命令
  - 对 `src/app/page.tsx` 和 `src/app/globals.css` 运行诊断检查，确保无新增错误。
  - 运行 `npm run lint`，若有结果，区分本次改动引入的问题与仓库既有 warning。
  - 如执行 `npm run build`，将既有数据类型失败与首页改动影响分开记录，不把无关失败误判为本次回归。
