## Proposed Changes

### 1. 新增第十四批英雄数据文件

- 新建 `src/data/champions/fourteenth-batch.ts`
- 只录入以下 5 个英雄：
  - `Pyke`
  - `Qiyana`
  - `Akali`
  - `Khazix`
  - `Zed`
- 每个英雄都要补齐：
  - `id / name / localizedName / roles / damagePatterns / combatTempos`
  - `scalingTags / triggerTraits / survivalStyles / utilityTags`
  - `abilityProfiles`
  - `positiveSignals / trapSignals`
  - `verification`
- 统一画像原则：
  - 主轴是 `机动进场 -> 压低血线 -> 收残局 / 再进场`
  - 需要明确区分：
    - `dash / blink / stealth / reposition`
    - `burst / pick / takedown / reset-like follow-up`
    - 先手开团型与纯收割型
  - 不能因为都是刺客就写成同一套模板

### 2. 各英雄画像落点

- `Pyke`
  - 重点机制：`hook / stun / camouflage / execute / reset-like ultimate follow-up`
  - 预期 archetype：机动收割 + 控制先手的混合体
  - 主要映射海克斯：`Executioner`、`Dashing`
  - 风险：不能被误写成纯辅助保人

- `Qiyana`
  - 重点机制：`dash / stealth / element swap / low-hp bonus / terrain cc`
  - 预期 archetype：高爆发 AD 机动刺客
  - 主要映射海克斯：`Dashing`、`Executioner`、`Thread the Needle`
  - 风险：元素机制与地形依赖不能被粗化成普通 burst assassin

- `Akali`
  - 重点机制：`stealth / dash / execute-like second R / energy-based extended skirmish`
  - 预期 archetype：高机动 AP 刺客
  - 主要映射海克斯：`Dashing`、`Executioner`
  - 风险：不能把她误归为纯法师或纯一次性爆发

- `Khazix`
  - 重点机制：`leap / stealth / isolation / takedown refresh follow-up`
  - 预期 archetype：单点击杀与残局收割
  - 主要映射海克斯：`Executioner`、`Dashing`
  - 风险：孤立无援与进化分支需要在 signals 中明确，否则解释力不够

- `Zed`
  - 重点机制：`shadow swap / mark burst / low-hp finishing / reposition`
  - 预期 archetype：技能型 AD 收割刺客
  - 主要映射海克斯：`Dashing`、`Executioner`
  - 风险：不能只写成“burst”，必须体现影子换位与二段收尾链路
