## Proposed Changes

### 1. 扩充第五批英雄画像

**文件**

- `src/data/champions/fifth-batch.ts`
- [champions/index.ts](file:///c:/Users/29403/Documents/trae_projects/hero-pro/src/data/champions/index.ts)

**做什么**

- 新增第五批英雄，名单固定为：
  - `Lulu`
  - `Swain`
  - `Vayne`
- 每个英雄继续沿用现有 `ChampionProfile` 结构，补齐：
  - 基础事实字段
  - `coreMechanics / antiPatterns`
  - `abilities[passive,q,w,e,r]`
  - `verification`

**为什么**

- `Lulu` 用来补强“给队友加成/护盾/保人”的附魔型辅助画像，能直接服务 `全心为你` 这类海克斯。

- `Vayne` 用来补强“单体持续普攻 + 位移拉扯 + 隐身自保”的射手画像，能提升普攻联动类海克斯的辨识度。

**怎么做**

- 继续使用 [sync-source-data.mjs](file:///c:/Users/29403/Documents/trae_projects/hero-pro/scripts/sync-source-data.mjs) 抓取双语英雄资料：
  - `npm run sync:sources -- --champions=Lulu,Swain,Vayne`
- 中文名、称号、技能名使用 `zh_CN` Data Dragon。
- 机制归纳以 `en_US` 技能文本为主，避免依赖社区二手解释。
- 在 `champions/index.ts` 中把第五批并入总数组，保持与前几批相同的显式导入方式。

### 2. 扩充第五批海克斯画像

**文件**

- `src/data/augments/fifth-batch.ts`
- [augments/index.ts](file:///c:/Users/29403/Documents/trae_projects/hero-pro/src/data/augments/index.ts)

**做什么**

- 新增第五批海克斯，名单固定为：
  - `Accelerating Sorcery / 加速巫术` (`id: 1`)
  - `All For You / 全心为你` (`id: 141`)
  - `Big Brain / 超强大脑` (`id: 180`)
  - `Typhoon / 台风` (`id: 87`)
- 每条继续补齐：
  - `summary / officialDescription`
  - `rewardTypes / triggerTypes / preferredUsers / playstyleFits / riskTags`
  - `positiveSignals / trapSignals`
  - `verification`

**为什么**

- `加速巫术` 适合拉开“普通法师”和“高频施法循环英雄”的差异。
- `全心为你` 能补上当前数据集中较弱的单体治疗/护盾强化方向。
- `超强大脑` 能把“法强转防御/护盾”的收益类型正式纳入画像与规则。
- `台风` 能增强对普攻频率、额外攻击目标和 `on-hit` 价值的识别。

**怎么做**

- 海克斯事实来源继续固定为三段式证据：
  - Riot Arena 模式说明
  - `source-cache/arena/15.11/en_us.json` 的客户端导出文本
  - `source-cache/arena/15.11/cherry-augments-en_gb.json` 与 `cherry-augments-zh_cn.json` 的条目名/稀有度/图标索引
- 中文名优先取本地缓存 `cherry-augments-zh_cn.json` 与 `zh_cn.json` 中可交叉验证的名称。
- `verification` 结构与前四批保持完全一致，不新增例外字段。

### 3. 精修推荐规则

**文件**

- [rules.ts](file:///c:/Users/29403/Documents/trae_projects/hero-pro/src/lib/recommendation/rules.ts)

**做什么**

- 在现有五个评分维度不变的前提下，补四类更细的机制加权：
  - `heal_shield` + `peel / ally_protect` 对辅助功能类海克斯的加成
  - `high_cast_frequency` + `haste / ap` 对高频施法类海克斯的加成
  - `attack / enhanced_attack / attack_speed` 对额外攻击目标、普攻联动类海克斯的加成
  - `ap` + `shield / frontline` 对“法强转护盾/容错”的生存型海克斯加成

**为什么**

- 当前规则已经能给出大方向，但对第五批这四条海克斯的“真正受益者”区分还不够细。
- 如果不补这一层，`Lulu / Swain / Vayne` 与已有样本之间的推荐差异会不够明显，用户只能看到“都像推荐”，看不出机制上的领先原因。

**怎么做**

- 不新增新 API，也不修改 `RecommendationResult` 结构。
- 只在现有规则分支中追加小范围显式条件：
  - `baseline_match` 补使用者画像细分
  - `trigger_reliability` 补高频施法与高频普攻稳定性
  - `reward_conversion` 补治疗护盾兑现、法强护盾兑现、额外目标普攻兑现
- 继续保持理由文案简洁，优先让 UI 能直接提取出“主要优势”和“主要风险”。

## Assumptions & Decisions

- “按你说的接着往下做”在当前仓库状态下，解释为：不重复第四批和滚动，而是直接进入第五批扩数与规则精修。
- 本轮继续坚持权威来源优先：
  - 英雄事实只以 Riot Data Dragon 为准
  - 海克斯事实以客户端导出文本 + Riot 模式说明 + CommunityDragon 条目索引交叉验证
- 本轮不新增后端、数据库、登录、埋点或运营后台。
- 本轮不扩展类型系统枚举；优先在现有 `types.ts` 能表达的范围内补样本和规则。
- 本轮目标总量固定为：
  - `15` 个英雄
  - `20` 个海克斯
