# 首页分区衔接箭头与滚动顺滑修复计划

## Summary
- 为首页第一大区与第二大区之间增加一个简洁的向下引导箭头，并配套渐变阴影/过渡层，让分区衔接更明确、更像完整的滚动叙事。
- 修复第二大区标题刚进入时“再往下滑会有一点滑不动”的问题，目标是在保留揭示仪式感的前提下，减少 sticky 区域带来的发涩卡段感。

## Current State Analysis
- 首页主结构位于 `src/app/page.tsx`，当前第一大区底部只有一段纯文本 `scroll-hint`，没有独立的视觉锚点或分区过渡组件。
- 第一大区与第二大区的衔接点在 `src/app/page.tsx` 中的 `home-first-region` 结束后直接进入 `query-reveal-region`。
- 滚动揭示逻辑在 `src/app/page.tsx` 中通过 `revealRegionRef + requestAnimationFrame + scroll/resize` 计算 `--query-reveal-progress`，动画强度由 CSS 变量驱动。
- 第二大区的发涩感主要来自 `src/app/globals.css` 中的 `.query-reveal-region { min-height: 185vh; }` 与 `.query-reveal-sticky { position: sticky; top: 0; min-height: 100vh; }` 组合：标题进入后，容器会在一整段 sticky 滚动里维持“锁住”的感觉。
- 当前已经存在移动端与 `prefers-reduced-motion` 的降级路径，后续调整不能破坏这两条路径。

## Proposed Changes
### `src/app/page.tsx`
- 在第一大区底部新增一个专门的衔接区块，替代当前单行 `scroll-hint` 文案。
- 衔接区块包含：
  - 简洁的向下引导箭头视觉元素
  - 轻量说明文案，继续承担“往下进入第二大区”的提示
  - 语义上保持为装饰性引导，不引入复杂交互或额外状态
- 保持第二大区主结构不变，但为后续 CSS 微调保留可挂载的类名/容器层，确保过渡层和箭头样式能独立控制。

### `src/app/globals.css`
- 为第一、第二大区之间新增“衔接带”样式：
  - 使用顶部到下方的渐变阴影/雾化过渡，让第一大区自然过渡到第二大区
  - 为箭头增加克制的品牌化视觉，方向明确但不过度抢戏
  - 可选轻微浮动或透明度呼吸，但默认强度要低，避免喧宾夺主
- 重构当前 `scroll-hint` 样式，使其演变为完整的分区过渡组件，而不是单独一行文字。
- 调整第二大区滚动揭示参数，减少“滑不动”感，同时保留仪式感：
  - 降低 `.query-reveal-region` 的总滚动距离，避免 sticky 区域过长
  - 调整 `.query-reveal-sticky` 的停驻策略，例如缩短锁定区、让内容更早开始接管视口
  - 优化标题区和查询面板的位移/透明度曲线，避免标题刚到位时用户滚动反馈不足
  - 必要时将揭示节奏改为“标题短暂停驻，面板更快接管”，而不是整个第二大区长时间吸附
- 同步更新移动端和 `prefers-reduced-motion` 分支：
  - 小屏下箭头与渐变阴影保留，但布局更紧凑
  - reduced motion 场景取消箭头漂浮类动态，只保留静态指引与顺序正确的内容流

### `src/components/query-panel.tsx`
- 预计不需要结构改动。
- 仅在第二大区衔接调整需要额外首屏占位或标题间距适配时，做最小样式配合，不改变查询逻辑和交互行为。

## Assumptions & Decisions
- “只因箭头”按“简洁下箭头”理解，不做玩梗化造型，保持产品整体克制风格。
- 滚动手感优先“保留仪式感但去掉明显发涩”，因此不会完全取消 sticky 揭示，只会缩短或优化其卡段区间。
- 本次范围只覆盖首页第一、第二大区衔接和第二大区起始滚动体验，不扩展到声明区或视频开场逻辑。
- 不新增第三方动画库，继续沿用现有 `React + CSS + scroll progress` 方案，降低实现复杂度和回归风险。

## Verification Steps
- 结构验证：
  - 确认第一大区底部出现新的箭头衔接带，而不是单行文本提示。
  - 确认第一大区、衔接带、第二大区的 DOM 顺序清晰，语义不混乱。
- 视觉验证：
  - 确认箭头方向明确，渐变阴影能自然连接两个大区，不会形成突兀的硬边。
  - 确认箭头与衔接带不会压住第一大区卡片，也不会抢过第二大区标题的视觉主次。
- 滚动验证：
  - 从第一大区底部继续下滑时，第二大区标题进入后不再出现明显“滑不动”或滚轮空转感。
  - 保留一定停驻揭示感，但标题和查询面板能更快接管视口。
- 降级验证：
  - 移动端下布局正常，没有箭头过大、阴影过重或 sticky 异常。
  - `prefers-reduced-motion` 下去掉动态扰动后，顺序和可读性保持正确。
- 工程验证：
  - 检查 `src/app/page.tsx`、`src/app/globals.css`、如有改动则包括 `src/components/query-panel.tsx` 的诊断信息。
  - 运行首页相关必要验证，确认未引入新的明显错误。
