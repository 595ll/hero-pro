# 查询区搜索筛选与多列布局计划

## Summary

- 目标：为查询区补上英雄搜索、海克斯搜索、海克斯等级筛选，并把英雄/海克斯选择区从“单列长列表”优化为更高密度的多列布局。
- 成功标准：
  - 英雄列表支持搜索，用户能快速定位英雄。
  - 海克斯列表支持搜索与稀有度筛选。
  - 海克斯筛选支持 `棱彩 / 黄金 / 白银` 多选并集，默认全开。
  - 英雄列表与海克斯列表均改为多格布局，在桌面端一次显示更多条目。
  - 不影响现有选择、重置、推荐排序、正式评分判断和结果展示。
- 约束：
  - 只改查询区前端交互与样式，不改 `src/lib/recommendation/*` 逻辑。
  - 不改现有数据模型与画像字段结构。
  - 保持当前图标接入、选中态和移动端兼容。

## Current State Analysis

### 当前查询区结构

- 查询主实现位于 `src/components/query-panel.tsx`。
- 核心交互状态当前只有：
  - `selectedChampionId`
  - `selectedAugmentIds`
- 当前英雄区：
  - 使用 `championProfiles.map(...)` 直接渲染按钮列表。
  - 容器是 `champion-scroll-area`，列表类名是 `champion-list`。
  - 每个英雄是单行 `champion-item`。
- 当前海克斯区：
  - 使用 `augmentProfiles.map(...)` 直接渲染按钮列表。
  - 容器是 `augment-scroll-area`，列表类名是 `augment-list`。
  - 每个海克斯是单行 `augment-item`。
- 当前结果区与推荐区都基于 `selectedAugmentIds` 和 `selectedChampionId` 工作，未依赖列表布局本身。

### 当前样式结构

- 查询区基础网格在 `src/app/globals.css` 的 `.query-grid`。
- 英雄列表和海克斯列表目前都是单列 `display: grid` + `gap`，但未配置多列模板。
- 已存在可复用样式：
  - `secondary-action-button`
  - `mini-tag`
  - `entity-icon*`
  - `field-head`
  - `champion-scroll-area`
  - `augment-scroll-area`
- 当前没有搜索框、筛选条、空结果提示样式。

### 当前可直接复用的数据

- 英雄搜索可直接基于 `championProfiles` 中的：
  - `name`
  - `title`
  - `id`
- 海克斯搜索可直接基于 `augmentProfiles` 中的：
  - `localizedName`
  - `name`
  - `summary`
- 海克斯等级筛选可直接基于 `augment.rarity`，已确认枚举值存在：
  - `silver`
  - `gold`
  - `prismatic`

## Assumptions & Decisions

- 用户目标是“更快找到条目 + 一屏看到更多条目”，不是重做整个结果区。
- 海克斯等级筛选采用：
  - 多选并集
  - 默认三档全开
  - 允许用户只勾选单档或任意组合
- 搜索采用前端即时过滤，不引入服务端或新数据源。
- 搜索匹配策略：
  - 英雄：`name / title / id`
  - 海克斯：`localizedName / name / summary`
  - 统一使用不区分大小写的包含匹配。
- 列表密度策略：
  - 英雄：桌面端 2 列，较窄屏幕退化为 1 列。
  - 海克斯：桌面端 2-3 列中的高密度方案，以当前面板宽度优先保证可读性；实现上优先采用 `repeat(auto-fill, minmax(...))`，让布局随宽度自适应。
- 已选中状态保持原逻辑：
  - 不额外置顶已选条目
  - 不改变已选海克斯排序
  - 仅通过过滤后的可见列表展示

## Detailed Interaction Design

### 英雄搜索

- 输入框放在英雄区标题与英雄列表之间。
- 占位文案示例：
  - `搜索英雄名、称号或英文 ID`
- 过滤后：
  - 只影响英雄按钮列表显示
  - 不自动清空当前已选英雄
  - 若当前已选英雄被搜索词过滤掉，详情卡与推荐逻辑仍保留，直到用户主动重置或改选

### 海克斯搜索

- 输入框放在海克斯区标题与筛选条上方或同一区块内。
- 占位文案示例：
  - `搜索海克斯名、英文名或效果摘要`
- 过滤后：
  - 只影响候选海克斯列表显示
  - 不自动移除已选海克斯
  - 已选海克斯若被过滤隐藏，结果区仍维持当前选择与推荐结果

### 海克斯等级筛选

- 使用三枚按钮实现，不用下拉：
  - `棱彩`
  - `黄金`
  - `白银`
- 默认三枚都激活。
- 点击行为：
  - 点击已激活项则取消该档
  - 点击未激活项则加入该档
- 防呆：
  - 若三档都被取消，建议实现上自动允许“空结果”，而不是强制回退；这样规则更直观

## Risks & Mitigations

- 风险：多列后海克斯卡片摘要过长，卡片高度差异明显。
  - 缓解：在多列布局下可考虑限制摘要展示行数或优化为更紧凑的纵向排版。
- 风险：搜索过滤后，用户误以为已选项被清空。
  - 缓解：结果区仍保留当前选择结果，且重置按钮行为不变。
- 风险：移动端筛选按钮拥挤。
  - 缓解：筛选条支持换行，按钮尺寸保持触控可点。
- 风险：英雄/海克斯多列后视觉噪声变高。
  - 缓解：优先保留图标、名称、少量摘要，不增加新的信息密度字段。
