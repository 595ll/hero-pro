## Proposed Changes

### 1. 扩充第六批英雄画像

**文件**

- `src/data/champions/sixth-batch.ts`
- [champions/index.ts](file:///c:/Users/29403/Documents/trae_projects/hero-pro/src/data/champions/index.ts)

**做什么**

- 新增第六批英雄，名单固定为：
  - `Soraka`
  - `Brand`
  - `Master Yi`

**为什么**

- `Soraka` 用来补强纯治疗/保人辅助画像，和 `Banner of Command`、`All For You`、`Sonic Boom` 的区分会更清晰。
- `Brand` 用来补强持续灼烧、多段命中法师画像，能更稳定地验证高频技能与持续伤害类海克斯。
- `Master Yi` 用来补强近战高频普攻/连续收割画像，适合承接 `Master of Duality`、`Combo Master` 一类高兑现普攻/连段海克斯。

**怎么做**

- 使用现有同步流程补抓双语英雄资料：
  - `npm run sync:sources -- --champions=Soraka,Brand,MasterYi`
- 继续沿用当前 `ChampionProfile` 结构：
  - `coreMechanics`
  - `antiPatterns`
  - `abilities`
  - `verification`
- 证据要求继续保持：
  - `verification.status = verified`
  - 至少含 `riot-ddragon`

### 2. 扩充第六批海克斯画像

**文件**

- `src/data/augments/sixth-batch.ts`
- [augments/index.ts](file:///c:/Users/29403/Documents/trae_projects/hero-pro/src/data/augments/index.ts)

**做什么**

- 新增第六批海克斯，名单固定为：
  - `Banner of Command / 号令之旗`
  - `Master of Duality / 二刀流`
  - `Spin To Win / 旋转至胜`
  - `Combo Master / 连招大师`

**为什么**

- `号令之旗` 能补强“附魔/保核/增益传递”方向，并和 `Soraka` 形成高价值基准样例。
- `二刀流` 能补强近战持续平 A / 双修成长方向，适合 `Master Yi` 一类英雄。
- `旋转至胜` 能补强近身持续战斗和高频输出方向，有利于拉开战士与法师差异。
- `连招大师` 能补强技能连段和连续命中的法师/刺客验证场景，也适合作为 `Brand` 的回归样例候选。

**怎么做**

- 继续使用现有 Arena 三段式证据：
  - `official-client`
  - `riot-support`
  - `community-mirror`
- 从 `source-cache/arena/15.11/en_us.json` 和 `cherry-augments-zh_cn.json` 双向核对：
  - `id`
  - `name`
  - `localizedName`
  - `rarity`
  - `officialDescription`
- `positiveSignals / trapSignals` 必须服务后续回归断言，不写泛化空话。

### 3. 扩充推荐回归基准

**文件**

- [audit-recommendations.mjs](file:///c:/Users/29403/Documents/trae_projects/hero-pro/scripts/audit-recommendations.mjs)
- [recommendation-audit.md](file:///c:/Users/29403/Documents/trae_projects/hero-pro/docs/recommendation-audit.md)

**做什么**

- 把当前 5 组基准扩到至少 8 组。
- 新增样例优先覆盖第六批英雄和第六批海克斯，而不是重复旧方向。

**为什么**

- 继续扩数据但不扩回归基准，会让“覆盖面”增长、“稳定性”不增长。
- 这轮的价值不是单纯把数量从 `15/20` 提到更高，而是让每新增一类 archetype 都进入可回归检查。

**怎么做**

- 新增至少 3 组断言：
  - `Soraka` 在 `Banner of Command / All For You / Typhoon` 中，优先 `Banner of Command` 或 `All For You`，并保证 `Typhoon` 不应排第一
  - `Brand` 在 `Combo Master / Accelerating Sorcery / All For You` 中，优先更偏技能链路的海克斯
  - `Master Yi` 在 `Master of Duality / Mystic Punch / All For You` 中，优先高频普攻/连战型海克斯
- 文档中同步更新新增基准样例，保持脚本和文档一致。

## Verification Steps

1. 同步第六批英雄源数据
   - `npm run sync:sources -- --champions=Soraka,Brand,MasterYi`
2. 录入第六批数据后跑结构审计
   - `npm run audit:data`
   - 预期输出：`Data audit passed: 18 champions, 24 augments`
3. 运行推荐回归校验
   - `npm run audit:recommendations`
   - 预期：至少通过 `8` 组场景
4. 运行静态检查
   - `npm run lint`
5. 运行生产构建
   - `npm run build`
6. 抽样人工复核
   - `Soraka` 的保人/增益海克斯不应被普攻海克斯压过
   - `Brand` 的技能链路海克斯应明显优于保人型海克斯
   - `Master Yi` 的高频普攻/持续作战海克斯应明显优于纯辅助型海克斯
